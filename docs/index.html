Onesait Platform Community es la distribución Open Source de Onesait Platform impulsada por Minsait.
Onesait Platform Community es una Plataforma Digital Open Source que habilita el desarrollo ágil de soluciones sobre tecnologías estándares y opensource y habilita el despliegue automático con Docker y Kubernetes sobre CaaS gestionando todo el ciclo de vida del producto.

Módulos Onesait Platform

  1. Onesait Platform Community es una plataforma para el desarrollo ágil de aplicaciones con despliegue continuo y multitenant.
  2. Incluye un Control Panel que permite gestionar todo el desarrollo, CI/CD y despliegue desde una consola web centralizada.
  3. Integra las mejores tecnologías open-source en su ámbito y hace de capa de Glue entre ellas para usarlas de una forma sencilla y segura
  4. Aisla de las complejidades de las tecnologías: Spring Boot, Kafka, MongoDB, Elastic, Spark, Docker, Kubernetes,....
  5. Se integra con plataformas CaaS como Rancher o Kubernetes y permite el despliegue en cualquier Cloud.
  6. También se integra con servicios PaaS de los principales proveedores (BigQuery, CosmosDB) permitiendo su uso integrado

  

onesait Platform Engine Modules

Spring
Swagger
Gravitee
API Manager
API Manager permite aplicar ontologías, flujos digitales y API swagger. Estas API se ven desde un portal y se exponen a través de OpenAPI. Extensión basada en Gravitee, para escenarios de apificación avanzados, para control de aceleración, políticas de seguridad avanzadas y otros protocolos.
Node Red
Flow Engine
Flow Engine le permite modelar la lógica empresarial a medida que el proceso fluye entre componentes que encapsulan la lógica. Node-red se utiliza como motor de ejecución.
Drool Tools
Rules Engine
El módulo Motor de reglas le permite crear reglas comerciales en la plataforma utilizando el lenguaje DRL (Drools Rule Language). Tecnología basada en Drools. Funciona con formato JSON tanto en entrada como en salida
Spring
oAuth2
Identity Manager
Todos los elementos de la plataforma estarán asegurados mediante configuración utilizando oAuth2 como estándar de seguridad. Permite la definición de Realms, Single Sign On, integración con LDAP y Active Directory y cifrado de datos y comunicaciones
Angular
Vue
Dashboards
Permite al usuario de la plataforma crear cuadros de mando visuales que explotan la información de la plataforma sea cual sea el repositorio, apoyando el desglose, permisos, visualización compleja de diferentes gadgets.
MongoDB
ElasticSearch
Postgres
Neo4j
Persistencia
Aísla la información de la plataforma del repositorio subyacente. El repositorio siempre ofrece una interfaz SQL de acceso unificado. Es compatible con varias tecnologías de bases de datos como MongoDB, Elasticsearh, Neo4j y bases de datos relacionales como Oracle, SQL Server, mysql, posgress, etc.
JSON
Fiware
GSMA
Semántica
Permite gestionar la semántica en la Plataforma de forma unificada y adapta el modelo semántico según el tipo de Modelo de Datos elegido por el proveedor de la solución. Se admite la semántica ligera basada en JSON (como GSMA, Fiware Data Model) y las transformaciones de otras semánticas (JSON-LD)
Spring
Spring Root
Microservicios
Soporte total para la creación de microservicios y gestión de su ciclo de vida completo, desde la compilación del código (basado en plantillas y arquetipos) hasta el despliegue en CaaS
WebSockets
Swagger
Kafka
Digital Broker
Permite que dispositivos, sistemas, APPs, etc., se comuniquen con la plataforma de forma bidireccional. Para ello, soporta varios protocolos incluyendo MQTT, REST, Web Sockets, Kafka, y ofrece APIs de cliente para diferentes lenguajes para que la comunicación con la plataforma sea lo más sencilla posible.

onesait Platform Intelligence Modules

Streamsets
DataFlow
Permite configurar flujos de datos entre fuentes y destinos + procesadores (plataforma, archivos, base de datos, ...) Es el mecanismo a utilizar cuando la plataforma va a recopilar información del sistema fuente y no el que la envía a la plataforma. La implementación de la plataforma se basa en componentes de Streamsets que permiten más de 100 conectores de origen / destino diferentes
Zeppelin
Notebooks
Esta herramienta permite a los científicos de datos explotar, visualizar y crear modelos y algoritmos sobre la información gestionada por la plataforma, de forma sencilla y con los principales lenguajes utilizados en ML, como Spark, R, Python, ... todo ello con la seguridad subyacente de la plataforma. Basado en Apache Zeppelin, podemos importar cuadernos de otras herramientas como Jupyter
Orange
Herramientas Guías de minería de datos
Permite a un usuario no experto realizar fácilmente análisis de datos interactivos, análisis de datos exploratorios, entrenar y ejecutar modelos o programación visual. El equipo de la plataforma recomienda utilizar Orange como herramienta de minería de datos, aprendizaje automático de código abierto y visualización de datos para principiantes y expertos. Flujos de trabajo de análisis de datos interactivos con una gran caja de herramientas
Azure
openCV
TensoFLow
Cognitiva
Conjunto de API para su uso en aplicaciones que requieren el uso de inteligencia artificial (IA). Permite consumir servicios de diferentes PaaS incluyendo entre otros: Lenguaje: análisis de texto, entidades, sentimiento; Traducción de textos; Análisis de imágenes incluyendo OCR, detección de entidades; y voz: habilitando TextToSpeech
Jasper
Report Engine
Los usuarios pueden adjuntar una plantilla de Jasper (formato .jrxml) creada en un editor de Jasper Reports como JasperSoft Studio. Adaptador de datos para conectarse a un servicio web REST. Este mecanismo es compatible con los adaptadores de datos JSON y XML y admite los verbos PUT, POST y GET.
Hadoop
Hive
Impala
DataHub
Este módulo utiliza Hadoop para almacenar, administrar y procesar grandes volúmenes de datos de manera eficiente y económica. Utiliza HIVE e Impala / Presto para consultas en línea

onesait Platform Things Modules

MQTT
OPC
IoT Broker
Componente que permite que los dispositivos y pasarelas se comuniquen con la plataforma de forma bidireccional. Para ello, soporta varios protocolos de IoT incluyendo MQTT, OPC, COAP, Kafka, y ofrece APIs de cliente para diferentes lenguajes para que la comunicación con la plataforma sea lo más sencilla posible.
Spring
Ansible
Swagger
Device Management
Una consola que permite administrar y operar los dispositivos de las soluciones de IoT. Edge Management System Cloud es una aplicación web que se ejecuta en un servidor Cloud que permite gestionar de forma remota todas las puertas de enlace Edge implementadas en el campo. Tiene una comunicación bidireccional con los dispositivos sin necesidad de la presencia física de un técnico para solucionarlo, permitiendo un control absoluto sobre los dispositivos de forma remota como si estuviéramos en el campo.
Spring
Svg
Synoptics
Enfocado en la creación de sistemas IoT en los que las capacidades Edge y Cloud de la plataforma puedan colaborar, permitiendo la comunicación bidireccional con dispositivos, modelado e implementación. Esta funcionalidad sinóptica es muy útil en escenarios industriales donde es necesario monitorear y actuar sobre señales físicas. Los sinópticos le permiten crear representaciones de tipo SCADA y dashbaord en tiempo real
Web of things
Digital twins
Este componente incluye un Digital Twin Broker, un modelado semántico basado en la API Web Thing y la generación de código ejecutable. Puedes interactuar y orquestar a los diferentes gemelos digitales registrados en la Plataforma desde Flow Engine
WIFI
Bluetooth
Zbee
Edge Engine
Estrategia híbrida para procesar datos cerca de donde se generan. Aumento de la velocidad, seguridad y reducción de costes. Arquitectura Edge Computing basada en contenedores, un conjunto de componentes técnicos que implementan esta arquitectura y un conjunto de herramientas para el desarrollo, despliegue y gobernanza de todo el ciclo de vida de los nodos computacionales y las aplicaciones en el edge.
Kubernetes
Spring
Workload Consolidation
Framework unificado para Industria IoT®, que es compatible con Edge Computing y Workload Consolidation Architecture (EWLC ™). Habilitación de modelos de aprendizaje automático contextuales con baja latencia y adquisición de automatización a través de redes RF y cableadas
